Модуль steelitem timeline
-------------
1. Объекты:
	1.1. Контроллер:
		- item_main.class.php
	
	1.2. Модель:
		- steelitem.class.php
	
	1.3. БД:
		- steelitem_timeline
		- sp_steelitem_timeline_get_list(param_steelitem_id INT), ORDER BY id DESC

2. Страница timeline
	2.1. Url:
		- /item/{id}/timeline
	
	2.2. Обработчик:
		- item_main.class.php::timeline(), доступ: ROLE_STAFF
	
	2.3. Контент:
		- Сетка:
		<table width="50%">
			{section}
			<tr>
				<td>
					<div class="timeline_{$steelitem_timeline.object_alias}">{$timeline_left[i]}</div>
				</td>
				<td style="padding-top:20px">
					<div class="timeline_{$steelitem_timeline.object_alias}">{$timeline_right[i]}</div>
				</td>				
			</tr>
			{/section}
		</table>
		
		- Блок события:
			<div class="timeline_{$steelitem_timeline.object_alias}">
				<h4>{$steelitem_timeline.created_at}<h4>
				описание
				<span>{$steelitem_timeline.modifier.login}</span>
			</div>
		
		- class="timeline_{$steelitem_timeline.object_alias}":
			- скругленные края
			- рамочка разного цвета для каждого типа события, cmr и ddt имеют одинаковый цвет
					
		- Возможные события:
			- Added to DB, ссылка на /item/{id}/view (ее нет пока). Для первой записи с object_alias = '' and object_id = 0.
			- Put into Incoming DDT {doc_no}, ссылка на /inddt/{steelitem_timeline.object_id}. Для записи с object_alias = 'inddt'.
			- Put into Order {doc_no}, ссылка на /order/{steelitem_timeline.object_id}. Для записи с object_alias = 'order'.
			- Put into RA {doc_no}, ссылка на /ra/{steelitem_timeline.object_id}. Для записи с object_alias = 'ra'.
			- Put into CMR {doc_no}, ссылка на /cmr/{steelitem_timeline.object_id}. Для записи с object_alias = 'cmr'.
			- Put into DDT {doc_no}, ссылка на /ddt/{steelitem_timeline.object_id}. Для записи с object_alias = 'ddt'.
			- Put into Invoice {doc_no}, ссылка на /invoice/{steelitem_timeline.object_id}. Для записи с object_alias = 'invoice'.
			- Put into Clame {doc_no}, ссылка на /invoice/{steelitem_timeline.object_id}. Для записи с object_alias = 'clame'.
			
	2.4. Функциональность:
		- полученный из $modelSteelItem->TimelineGetList список должен быть дополнен cmr или ddt, в котором находится айтем для каждого object_alias = 'ra'
		- список событий разбивается по принципу первай-второй, на два списка: timeline_left, timeline_right
		- section делается по наиболее полному списку из timeline_left, timeline_right
